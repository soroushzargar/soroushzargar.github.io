Coverage is a promise,
not a metric.

By
Soroush H. Zargarbashi
Published
April 14, 2026
Filed under
Theory · Manifesto
Set in
Cologne, DE

Most machine learning metrics describe the past. Accuracy says: out of the inputs we've already seen, this is the fraction we got right. Coverage is different. Coverage is a promise about the future — about every input the model hasn't seen yet.

There is a small but stubborn family of methods in statistics that does not ask you to trust the model. It asks the model to declare, in advance, how confident it is — and then it checks the receipts. The cleanest member of that family is conformal prediction, and once you have seen it, you cannot un-see it.

This essay is an attempt to explain why I think coverage is the most underappreciated idea in modern machine learning. It is not new — Vladimir Vovk and his collaborators were writing about it in the 1990s — but it has acquired, in the past five years, a strange and welcome second life. I want to argue for why.

What coverage is, exactly

Suppose your classifier is asked to label an image as one of K classes. The usual output is a single label and a softmax-shaped lie about confidence. A conformal procedure outputs something different: a set of labels. The set is constructed so that for every new test point, the probability that it contains the true label is at least 1 − α. You pick α; the procedure delivers the guarantee.

ℙ ( ytest ∈ C(xtest) ) ≥ 1 − α

The catch — and it is a small one — is exchangeability. Your calibration data and your test data must be drawn from the same distribution (or rather, be jointly exchangeable, which is a weaker condition). Get that, and the guarantee is distribution-free in the technical sense. It does not care what model you used. It does not care whether the model is well-calibrated. It does not even care whether the model is any good.

Conformal prediction is the rare statistical method that improves the worst model in your portfolio by exactly as much as it improves the best.

Why this changes the question

When you frame uncertainty as a number — 0.93, say — you are implicitly asking the model to be a thermometer. The number tells you how hot it is. You trust the thermometer because you trust the calibration of mercury and glass. With a deep network, you have neither the mercury nor the glass. You have a function trained to minimize cross-entropy, which is to say, a function trained to be confident. The Brier score is a downstream side-effect; calibration is rarely an explicit objective.

Set-valued prediction reframes the question. The model is no longer asked, "how confident are you?" It is asked, "what would you bet on?" And — crucially — it pays a cost when it bets on more than it has to. |C(x)| is the size of the prediction set; smaller is better, but only if the coverage holds. The trade-off is honest and visible.

A 90% set is a contract with reality. The fine print is short. The signatures are mathematical.

The trick, in one paragraph

Take a model. Take a calibration set you held out. Compute a non-conformity score for every calibration point — anything that quantifies how badly the model would have failed on it, e.g. one minus the softmax of the true class. Sort the scores. Find the (1−α)-quantile. At test time, include in the prediction set every class whose score is below that threshold. That's it. That is the whole method. It runs in a few lines of code, and it produces sets with finite-sample, distribution-free coverage. The proof fits on a napkin. 

# pseudo-code, the whole thing
scores = [score(x_i, y_i) for (x_i, y_i) in cal]
q = quantile(scores, level=1-alpha)
C = lambda x: {y for y in classes if score(x, y) <= q}

When I show this in a talk, someone always asks: where is the catch? The catch is that the sets can be large when the model is bad. That is not a bug. That is the model telling you it is bad. Most ML pipelines have no mechanism for this confession. Conformal prediction is, among other things, a confessional booth.

Where this gets hard

Exchangeability is the catch. Real test data drifts. Graphs have dependencies. LLMs have prompts that look nothing like their training. Adversaries do things on purpose. Most of my own research is, in one form or another, an attempt to weaken or replace the exchangeability assumption — to keep the coverage guarantee alive when the world is being uncooperative.

Three lines of attack interest me:

  • Graphs. When your data points are nodes in a network, they are not i.i.d. and not exchangeable. Our 2023 paper gave the first principled fix; the 2025 follow-up extended it to inductive settings under distribution shift.
  • Robustness. If an adversary perturbs the test input, the calibration data is no longer exchangeable with what the model sees. We can recover guarantees by considering the worst-case score over the perturbation set — and we can do it without blowing up set sizes.
  • LLMs. Sequence outputs are not labels. Coverage for them has to be redefined — over factual claims, over abstention behaviors, over the answer space itself. This is the next frontier and most of it is unwritten.

What I want you to take away

If you are deploying ML and you are not measuring some form of coverage — even a rough one, even on a held-out slice — you are guessing. Accuracy on a benchmark is not the same as honesty in production. A prediction set is a contract. A point prediction is a marketing claim.

The good news is that the entry cost is small. You can wrap conformal prediction around any model you already have. You do not have to retrain. You do not have to switch frameworks. You just have to hold out a calibration set — the same calibration set you should have been holding out anyway — and write the four lines above.
